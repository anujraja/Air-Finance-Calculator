How it works

flowchart TB
  U["Onboarding wizard<br/>5 short steps"] --> V{"Zod schema<br/>validation"}
  V -->|invalid| U
  V -->|valid| E["Calculation engine<br/>pure, tested functions"]
  E --> T["Income & tax<br/>Federal + Ontario · CPP / EI"]
  E --> C["Corporate tax<br/>small-business rate"]
  E --> A["Affordability<br/>GDS / TDS + stress test"]
  E --> S["Down-payment plan<br/>timeline + projection"]
  E --> M["Target home<br/>mortgage + total cost"]
  T --> D["Analysis dashboard"]
  C --> D
  A --> D
  S --> D
  M --> D

Every number the dashboard shows comes from one set of pure functions, each covered by unit tests.

sequenceDiagram
  participant U as You
  participant C as Client · Zod
  participant S as /api/analyze
  U->>C: Fill five short steps
  C->>C: Validate instantly — no bad request leaves
  C->>S: POST validated profile
  S->>S: Re-validate + compute
  S-->>C: Full analysis
  C-->>U: Render dashboard + charts

The same schema guards the input on the client and the server; an isomorphic fallback keeps results honest if the request fails.
